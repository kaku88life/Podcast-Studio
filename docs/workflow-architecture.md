# Podcast 內容工廠 — 工作流架構文件

> 以「兩人自然閒聊式對話」為核心，用 AI 放大產能、降低後製摩擦，把「發想 → 錄製 → 產出 → 後製 → 推廣」整合成穩定的內容生產線。
>
> 文件日期：2026-06-14 / 提案專案名：podcast-content-factory（可改）

---

## 一、核心理念

> **自然閒聊對話是「原料」，AI 是「放大器」，不是取代者。**

兩位主持人針對有價值的題目自然討論——這份真實的思考與默契是內容的靈魂。AI 負責把這份原料**整理得更流暢、產出得更省力**，而不是憑空捏造觀點。所有 AI 環節都保留「人在迴路（human-in-the-loop）」的覆核點。

### 兩大痛點 → 兩大解法

| 痛點 | 解法 | 對應階段 |
|------|------|---------|
| 真人出鏡成本高（服裝/光線/畫面） | **視覺生成**：虛擬形象 / 簡報動畫 / 圖表 / B-roll，不必出鏡 | 階段 4 |
| 正式錄製緊張卡詞、剪輯耗時 | **語音合成**：先輕鬆討論，再用克隆聲音依腳本生成流暢對話 | 階段 2-3 |

---

## 二、為什麼另開專案

- 現有 `line-inspiration-helper` 是個人語音工具，定位、使用者、生命週期都不同。
- 內容工廠是橫跨「選題→發佈」的產線，範圍與複雜度高一個量級。
- **複用而非混入**：把現有的轉錄 / 字幕 / 詞庫 / 波形編輯器當「捕捉層」與「組裝層」的元件，以模組或服務形式引用。

## 三、端到端工作流（10 階段）

```
階段 0  選題發想 Ideation
        題庫 / 選題評分(價值×熱度×可做性) / 參考資料蒐集 / 大綱
        [人主導] AI 輔助找題、查資料

階段 1  討論捕捉 Capture                     ← 低壓力，這是內容靈魂
        兩人自然閒聊(不出鏡、不怕卡詞) → 錄音 → 轉錄 → 主稿+字幕+音檔
        [複用 line-inspiration-helper 的轉錄/字幕/詞庫]

階段 2  腳本化 Scripting                      ← 解決卡詞、省剪輯
        逐字稿 → AI 整理成結構化對話腳本(去卡詞/贅詞/重複、補邏輯、保留口語風)
        [人在迴路] 人工覆核定稿，確保觀點正確

階段 3  語音合成 Voice Synthesis              ← 解決卡詞、流暢度
        克隆的兩人聲音 + 腳本 → TTS 生成自然對話音檔(無卡詞)
        [人在迴路] 試聽、調語氣情緒

階段 4  視覺生成 Visuals                       ← 解決出鏡成本
        不出鏡：虛擬形象 / 簡報動畫 / 字幕卡 / B-roll / AI 圖
        [人補充] 數據圖表、資料來源、案例圖片(品質與可信度關鍵)

階段 5  影片組裝 Assembly
        音檔 + 字幕 + 視覺 → 影片(Remotion 模板自動化 / 波形編輯器手動精修)

階段 6  後製 Post-production
        片頭片尾 / BGM / 轉場 / 品牌包裝(模板化)

階段 7  多平台轉化 Repurpose
        一份 master → YouTube 長片 / Shorts / Podcast 音檔 / 部落格 / 社群短文

階段 8  發佈推廣 Publish
        多平台排程 / 標題 / 縮圖 / SEO / 社群推廣

階段 9  資產管理 Asset Management              ← 貫穿全程
        主稿/音檔/字幕/圖表/影片版本/聲音模型 集中管理、可重用、可回溯
```

**人工介入點（品質防線）**：階段 0 選題、階段 2 腳本定稿、階段 4 數據圖表、階段 5/6 成片覆核。其餘盡量自動化。

## 五、Episode 資料模型（貫穿全程的單集物件）

```jsonc
Episode {
  id, title, topic, status,           // status: ideation→captured→scripted→...→published

  ideation: {
    references: [{title, url, note}],
    outline: string,
    score: {value, trend, feasibility}
  },

  capture: {                          // 複用現有轉錄/字幕
    raw_audio_url, transcript,
    segments: [{source_start, source_end, text, speaker}]
  },

  script: {                           // AI 腳本化 + 人工定稿
    dialogue: [{speaker, text, note}],
    approved: boolean, approved_by
  },

  voice: {                            // 克隆聲音合成
    lines: [{speaker, text, audio_url, voice_id, emotion}],
    master_audio_url
  },

  visuals: {                          // 不出鏡的視覺素材
    slides: [{time, type, content}],
    charts: [{time, data_source, image_url}],   // 人工補
    images: [{time, prompt|file, credit}],
    broll: [{time, clip_url}]
  },

  assembly: {                         // 複用波形編輯器 EDL
    timeline_edl, video_drafts: [{resolution, url}]
  },

  outputs: {
    youtube: {video_url, title, desc, chapters},
    shorts: [{url, hook}],
    podcast_audio_url, blog_md, social: [{platform, text}]
  },

  publish: { platform_status: {youtube, spotify, ig, fb, threads} }
}
```

## 六、技術選型（含權衡與成本概估）

### 語音克隆 / TTS（中文 + 自訂聲音）
| 方案 | 中文 | 自訂克隆 | 成本 | 備註 |
|------|------|---------|------|------|
| **ElevenLabs** | 良 | 有 | ~$22/月起(依字數) | 最自然、商用 OK、API 成熟 → **首選** |
| Fish Audio | 優 | 有 | 便宜 | 中文表現好 |
| GPT-SoVITS | 優 | 有 | 免費(需自架 GPU) | 開源、可完全掌控、學習曲線高 |
| MiniMax / 火山引擎 | 優 | 有 | 中 | 中文 TTS 強 |
| OpenAI TTS | 良 | 無 | 低 | 只有預設聲音，不能克隆 → 不適用 |

### 視覺生成
| 方案 | 用途 | 成本 | 備註 |
|------|------|------|------|
| **Remotion** | 簡報/動態圖文式影片 | 免費(自架渲染) | React 寫影片、程式化、模板化 → 知識型 podcast 首選 |
| HeyGen / D-ID | 虛擬主播(會動人像) | $$$ | 想要「有人」但不真出鏡 |
| Flux / DALL-E / Midjourney | 配圖、封面、案例圖 | 低-中 | AI 圖 |
| 圖庫 / 自製圖表 | 數據、可信度 | 人力 | 人工補，品質關鍵 |

### 影片組裝 / 編排
| 方案 | 用途 |
|------|------|
| Remotion | 模板化自動組裝(音檔+字幕+視覺) |
| 波形編輯器(本系列另一文件) | 需要手動精修時 |
| FFmpeg | 底層渲染、轉碼、多平台格式 |
| 自建 pipeline + job queue / n8n | 串接各階段 API、編排自動化 |

### 後端 / 基礎
- Python(FastAPI) 或 Node — 沿用你熟悉的 Python。
- PostgreSQL(Episode 狀態) + 物件儲存(素材) + Redis(任務佇列)。

## 七、語音克隆的正當性與合規

- **克隆「自己的聲音」完全正當**——這是創作者常見做法（用自己的聲音提升產能），非冒充他人。
- 前提：取得學弟**明確同意**克隆他的聲音（你們是合作夥伴，書面記一筆即可）。
- 平台揭露：YouTube 等對「合成/變造內容」有揭露選項，照實勾選即合規，不影響觸及。
- 守住底線：只生成「兩人真實討論過、且已覆核」的觀點，不無中生有、不假造數據——這也是品質與信任的根本。

## 八、MVP：最小可行的半自動工廠

先把摩擦力最大的環節自動化，跑通一集端到端：

```
1. 輕鬆討論錄音 → 轉錄 + 字幕          [複用既有，幾乎 0 開發]
2. AI 腳本化(LLM 整理對話) → 人工定稿   [新，1 個 LLM 呼叫 + 編輯介面]
3. ElevenLabs 雙聲音合成               [新，串 API]
4. Remotion 簡報式影片模板             [新，音檔+字幕+圖表自動排版]
   (音檔 + 字幕 + 你補的圖表)
5. 人工覆核 → 匯出 YouTube 長片
```

跑通這條鏈，後製時間可從「數小時」降到「組裝 + 覆核」的等級，且品質可控。之後再逐步補：Shorts 自動切、多平台發佈、虛擬主播、自動配圖。

## 九、分階段路線圖

| 階段 | 目標 | 產出驗收 |
|------|------|---------|
| **M1 捕捉複用** | 把既有轉錄/字幕接成工廠的「捕捉層」 | 一集討論 → 主稿+字幕+音檔 |
| **M2 腳本化** | LLM 把逐字稿整理成對話腳本 + 覆核介面 | 卡詞/贅詞清除、保留口語風、人工定稿 |
| **M3 語音合成** | 克隆雙聲音、依腳本生成 master 音檔 | 自然無卡詞的對話音檔 |
| **M4 影片組裝** | Remotion 模板(音+字幕+圖表) → YouTube | 第一支半自動成片 |
| **M5 多平台轉化** | 一份 master → Shorts/音檔/部落格/社群 | 一鍵多格式 |
| **M6 發佈與素材庫** | 多平台排程 + 資產管理 | 排程發佈 + 素材可重用 |

## 十一、風險與權衡

| 風險 | 對策 |
|------|------|
| AI 語音「恐怖谷」、聽起來假 | 選 ElevenLabs/Fish 等高品質方案；保留真實口語停頓/語氣；必要時混真人錄音 |
| 內容失真 / 觀點被 AI 帶偏 | 嚴守人工定稿關卡；AI 只整理不創造觀點 |
| 全自動導致同質化、失去個人風格 | 核心永遠是兩人真實討論；模板保留品牌與個性 |
| 語音/視覺 API 成本累積 | 先 MVP 控規模；批次處理；長內容才用重武器 |
| 平台對 AI 內容政策變動 | 照實揭露；維持「真實討論為本」的內容正當性 |
| 過度工程、遲遲不產出 | 先跑通 MVP 一集，再迭代；別一開始就建全自動工廠 |
