# 蒸餾 00 — 一頁總結

> 2026-07-05｜Podcast Studio 蒸餾（只到 spec/roadmap）。三份詳版：[01 現況真相](01-現況真相.md)、[02 目標spec與願景](02-目標spec與願景.md)、[03 真backlog](03-真backlog.md)。

## 現在站在哪
- **做出來的**：製作管理中心 MVP——三個即時共編面板（看板/工具表/準備清單）＋ Supabase 後端 ＋ Google Drive A–G 結構 ＋ GAS 腳本。全在 `feat/production-hub` 分支（**PR #1 未 merge**）。
- **還沒做的**：真正的內容產線（腳本化→配音→影片）全部只是規劃，一行產線程式碼都還沒有。
- **架構很完整**：pipeline、Episode 資料模型、4 個人工覆核閘門、`cue_id` 綁句時間軸、工具選型傾向都寫清楚了，可直接照著開發。

## 接下來三步（照做就對）

**第 1 步：把樞紐先跑掉（人，不寫程式）**
Hiro 簽聲音克隆書面同意書 → 兩人各錄 3–5 分鐘乾淨語料 → 丟 ElevenLabs 建模試聽。
👉 聲音品質過不過關，決定整條線可不可行。這步**沒有程式，便宜模型也幫不上**，卡在「要先做、要先簽」。

**第 2 步：收尾現有成果（便宜就夠）**
Merge PR #1 回 main、Drive 收尾（拖 PNG／跑 GAS／分享給 Hiro）、和 Hiro 對齊 11 個待確認問題、開始用工具確認表試工具。
另外兩件小事：**確認 6/7 試錄到底錄了沒**（錄了的話音檔＝捕捉層第一份測試素材）；**整理兩份會議逐字稿**（5/23 京都物件那份是現成的首集素材）。
👉 幾乎都是手動操作與兩人會議，程式部分 💚 **便宜模型/便宜 agent 就夠**。

**第 3 步：搭 MVP 骨幹（大多便宜，只有一處要貴模型）**
接捕捉層（複用 line-inspiration-helper）→ 腳本化 LLM＋覆核介面 → ElevenLabs TTS 串接 → Remotion 影片模板。
👉 串接、前端、樣板都 💚 **便宜模型可承接**；唯一要 🔶 **Opus** 的是「**首集腳本的語感校準**」（守口語風＋不幫你們捏造沒講過的觀點），校準完之後量產降 Sonnet。

## 哪步便宜模型就夠？（一句話）
**真正需要貴模型的只有「第一集腳本的語感校準」一處；其餘不是人的決定（同意書/試聽/選題/命名），就是便宜模型足以承接（串接/前端/樣板/排程）。**

## 三條紅線（聲音克隆合規，別漏）
1. 克隆 Hiro 聲音前，**書面同意書**要先簽（存 Drive 法務夾）。
2. 每次發佈**強制勾平台 AI 揭露**（YouTube/IG），描述加一句合成聲明。
3. 只做「兩人真的討論過、已覆核」的內容，不捏造數據。
